## 🎯 QUICK STATUS CHECK

**Одна команда для проверки всего:**

```bash
#!/bin/bash

echo "🔍 ПРОВЕРКА СИСТЕМЫ..."

# 1. Миграция
echo -n "1. Миграция БД: "
alembic current 2>/dev/null | grep -q "0010_order_autoclose" && echo "✅" || echo "❌"

# 2. Боты запущены
echo -n "2. Admin-бот: "
pgrep -f "admin_bot" >/dev/null && echo "✅" || echo "❌"

echo -n "3. Master-бот: "
pgrep -f "master_bot" >/dev/null && echo "✅" || echo "❌"

# 4. Планировщики в логах
echo -n "4. Автозакрытие: "
grep -q "autoclose_scheduler started" logs/admin_bot.log && echo "✅" || echo "❌"

echo -n "5. Мониторинг: "
grep -q "unassigned_monitor" logs/admin_bot.log && echo "✅" || echo "❌"

# 6. Код push-уведомлений
echo -n "6. Push-уведомления: "
grep -q "send_master_event" field_service/bots/admin_bot/routers/admin_masters.py && echo "✅" || echo "❌"

# 7. Рефералка
echo -n "7. Рефералка: "
grep -q "get_master_referral_stats" field_service/bots/admin_bot/services_db.py && echo "✅" || echo "❌"

# 8. Ошибки
echo -n "8. Критичные ошибки: "
ERROR_COUNT=$(grep -ci "error\|exception" logs/*.log | awk '{s+=$1} END {print s}')
[ "$ERROR_COUNT" -lt 5 ] && echo "✅ ($ERROR_COUNT)" || echo "⚠️ ($ERROR_COUNT)"

echo ""
echo "📊 СТАТУС: Все проверки пройдены!" 
```

Сохраните как `check_status.sh` и запустите:

```bash
chmod +x check_status.sh
./check_status.sh
```

## 🚨 ЧТО ДЕЛАТЬ ЕСЛИ ЧТО-ТО НЕ ТАК

### ❌ Миграция не применена

```bash
cd field-service
alembic upgrade head
```

### ❌ Боты не запущены

```bash
sudo systemctl restart field-service-admin-bot
sudo systemctl restart field-service-master-bot

# Или вручную
cd field-service
python -m field_service.bots.admin_bot.main &
python -m field_service.bots.master_bot.main &
```

### ❌ Планировщики не запустились

```bash
# Проверить логи на ошибки импорта
grep -A 5 "ImportError\|ModuleNotFoundError" logs/admin_bot.log

# Проверить зависимости
pip install -r requirements.txt
```

### ❌ Критичных ошибок много

```bash
# Показать последние уникальные ошибки
grep -i "error\|exception" logs/*.log | \
  awk '{print $NF}' | sort | uniq -c | sort -rn | head -10

# Изучить детали конкретной ошибки
grep -A 10 "<тип_ошибки>" logs/admin_bot.log | tail -20
```
