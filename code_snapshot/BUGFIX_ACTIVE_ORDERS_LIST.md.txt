# 🔧 BUGFIX: Список активных заказов и переход после принятия

**Дата:** 2025-10-10  
**Приоритет:** P0 (Критический UX)

## 📝 Описание проблем

### Проблема 1: Неправильный переход после взятия заказа
После того как мастер принимает заказ, бот показывает сообщение "Заявка принята. Удачи в работе!" но остаётся на экране "Новые заявки". Мастеру приходится вручную переходить в "Активный заказ" чтобы увидеть принятый заказ.

**Ожидаемое поведение:** После принятия заказа бот должен автоматически перейти к карточке этого заказа в разделе активных заказов.

### Проблема 2: Показывается только последний активный заказ
В разделе "Активный заказ" показывается только последний взятый заказ, хотя у мастера может быть несколько активных заказов одновременно. Предыдущие заказы "висят" на мастере, но не отображаются в интерфейсе.

**Ожидаемое поведение:** 
- Раздел должен называться "Активные заказы" (множественное число)
- Должен показываться список ВСЕХ активных заказов мастера
- Каждый заказ должен иметь кнопку для открытия его карточки

### 4. Переработана функция отображения активных заказов
