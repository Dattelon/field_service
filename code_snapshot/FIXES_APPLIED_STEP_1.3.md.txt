### ✅ Исправление 1.3: Гарантийные заказы с preferred мастером
**Файл:** `field_service/services/distribution_scheduler.py`
**Строки:** ~348-440, ~786-840

**Проблема:**
```python
# До исправления:
preferred_id = order.preferred_master_id if order_kind == "GUARANTEE" else None
ranked = await _candidates(..., preferred_mid=preferred_id)

# Если preferred мастер:
# - Не на смене (is_on_shift = FALSE)
# - На перерыве (break_until > NOW)
# - Превышен лимит (active_cnt >= max_limit)
# → ranked = [] → ЭСКАЛАЦИЯ к логисту
# → Другие доступные мастера ИГНОРИРУЮТСЯ!
```

**Решение:**
1. **Добавлена диагностическая функция** `_check_preferred_master_availability()`
   - Проверяет все условия доступности мастера
   - Возвращает детальные причины недоступности
   - Логирует статус preferred мастера

2. **Изменена логика распределения:**
   ```python
   # Шаг 1: Диагностика preferred мастера
   if preferred_id:
       diag = await _check_preferred_master_availability(...)
       if not diag["available"]:
           logger.info("preferred_master UNAVAILABLE: %s", diag["reasons"])
   
   # Шаг 2: Поиск с preferred приоритетом
   ranked = await _candidates(..., preferred_mid=preferred_id)
   
   # Шаг 3: Fallback на всех мастеров если preferred недоступен
   if preferred_id and not ranked:
       ranked = await _candidates(..., preferred_mid=None)
       if ranked:
           logger.info("found %d alternative masters", len(ranked))
   
   # Шаг 4: Эскалация ТОЛЬКО если нет кандидатов вообще
   if not ranked:
       # Escalate to logist
   ```

### `_check_preferred_master_availability()`

**Проверяемые условия:**
- ✅ `is_active` - Мастер активен
- ✅ `is_blocked` - Не заблокирован

- ✅ `break_until` - Не на перерыве

- ✅ `has_district` - Работает в районе
