## 📋 ВЫПОЛНЕННЫЕ ИСПРАВЛЕНИЯ

### ✅ Исправление 1.1: Race Condition в offer_accept()
**Файл:** `field_service/bots/master_bot/handlers/orders.py`
**Строки:** ~196-215

**Проблема:**
- Проверка и обновление заказа не были атомарными
- Два мастера могли одновременно принять один заказ

**Решение:**
```python
# ✅ Добавлена атомарная блокировка
order_snapshot = await session.execute(
    select(m.orders.status, m.orders.assigned_master_id, m.orders.version)
    .where(m.orders.id == order_id)
    .with_for_update(skip_locked=True)  # ← КЛЮЧЕВОЕ ИЗМЕНЕНИЕ
    .limit(1)
)
```

**Эффект:**
- ✅ Предотвращена двойная блокировка заказов
- ✅ Если заказ уже заблокирован - мастер получит сообщение "Заказ уже взят"
- ✅ Нет ожидания - skip_locked немедленно возвращает None

### ✅ Исправление 1.2: Разрешение DEFERRED заказов
**Файл:** `field_service/bots/master_bot/handlers/orders.py`
**Строки:** ~226-237

**Проблема:**
- DEFERRED заказы блокировались для принятия
- Мастер не мог принять оффер в нерабочее время
- Заказ "висел" до возобновления рабочего времени

**Решение:**
```python
# ✅ DEFERRED добавлен в разрешённые статусы
allowed_statuses = {
    m.OrderStatus.SEARCHING,
    m.OrderStatus.GUARANTEE,
    m.OrderStatus.CREATED,
    m.OrderStatus.DEFERRED,  # ← КЛЮЧЕВОЕ ИЗМЕНЕНИЕ
}

# Логирование принятия DEFERRED
if current_status == m.OrderStatus.DEFERRED:
    _log.info(
        "offer_accept: accepting DEFERRED order=%s by master=%s",
        order_id, master.id
    )
```

**Эффект:**
- ✅ Мастер может принять оффер в нерабочее время
- ✅ Заказ автоматически переходит DEFERRED → ASSIGNED
- ✅ Работа начинается сразу после принятия

### ✅ Исправление 1.2.1: Поддержка DEFERRED в распределении
**Файл:** `field_service/services/distribution_scheduler.py`
**Строки:** ~133-165

**Проблема:**
- DEFERRED заказы не попадали в выборку для распределения
- Существующие офферы не учитывались

**Решение:**
```sql
-- ✅ Обновлённый SQL запрос
WHERE (
    o.status IN ('SEARCHING','GUARANTEE')
    OR (
      o.status = 'DEFERRED' 
      AND EXISTS (
        SELECT 1 FROM offers 
        WHERE order_id = o.id 
          AND state IN ('SENT', 'VIEWED')
      )
    )
  )
AND o.assigned_master_id IS NULL
```

**Эффект:**
- ✅ DEFERRED заказы с активными офферами учитываются
- ✅ Мастера видят все доступные офферы, даже в DEFERRED
- ✅ Распределение работает корректно в нерабочее время
